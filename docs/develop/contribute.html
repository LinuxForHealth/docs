Contribute¶

Contributions of all kinds are welcome and encouraged.

To contribute code or documentation for an existing issue, please open a pull request.

A good way to get started and familiarize yourself with the code base is to address issues labeled good first issue.

Proposing new features¶

If you would like to contribute a new feature, please open an issue prior to opening a pull request.

Fixing defects¶

If you encounter and/or would like to fix a defect, please open an issue if there isn’t one already open before opening a pull request.

Coding style guidelines¶

If you decide to contribute code, please help keep our code base clean and stable. Coding styles are not strictly enforced, but please considering the following guidelines:

  1. Include good unit test coverage in your pull requests.

  2. Make sure your code is legible and there are sufficient comments such that developers can easily read and understand the code.

  3. Contribute corresponding documentation pull requests as appropriate to keep the project well documented.

Project committers will review code updates within pull requests and suggest improvements as necessary.
